I've added a WebConfig class to enable CORS policy for your API. This configuration allows cross-origin requests from any origin to all /api/** endpoints, which is useful for frontend applications running on different domains or ports.
The configuration:
Allows all origins (*)
Permits common HTTP methods (GET, POST, PUT, DELETE, OPTIONS)
Allows all headers
Disables credentials (since origins are wildcard)
If you need more restrictive CORS settings (e.g., specific allowed origins), you can modify the allowedOrigins() method accordingly. The application should now accept CORS requests from web browsers.
